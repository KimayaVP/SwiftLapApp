## What this is

Native **SwiftUI** apps that mirror the SwiftLap web app, for swimmers and coaches:
- **iOS app** (`SwiftLap/` target) â€” full swimmer + coach experience.
- **watchOS app** (`Watch/` target) â€” swim tracker, **embedded in the iOS app** but **independent** (`WKRunsIndependentlyOfCompanionApp`): ships in one App Store listing, installs onto the paired watch with the iPhone app, yet still runs without it. (Migrated from the old `SwiftLapWatch` repo, which this supersedes.)

Both are clients of the **existing Express/Supabase backend** at `https://swiftlap.onrender.com` (repo: `KimayaVP/SwiftLap`, sibling folder `../SwiftLap`). We did **not** rewrite the backend â€” the apps call the same REST API the website does.

GitHub: `KimayaVP/SwiftLapApp` (public), branch `main` (work is pushed directly to main here).

## Cross-platform parity (important)

Any feature/change must be replicated across **web + iOS (+ future Android)** â€” never ship to one client only. Backend endpoints are usually added once; the parity work is per-client UI.

## Build tooling (READ THIS)

The project is **generated by XcodeGen** from `project.yml`. The `.xcodeproj`, `SwiftLap/Info.plist`, `SwiftLap/SwiftLap.entitlements`, `Watch/Info.plist`, `Watch/SwiftLapWatch.entitlements` are **generated and git-ignored** â€” never hand-edit them; edit `project.yml` and regenerate.

XcodeGen is **not on PATH / not via brew** (brew lacked write perms). It is a **local binary**:
`/Users/Kimaya/SwiftLap-project/.tools/xcodegen/bin/xcodegen`

Common commands (run from this folder):
```bash
# Regenerate the project after adding files or editing project.yml
../.tools/xcodegen/bin/xcodegen generate

# Build the iOS app for the simulator (no signing needed)
xcodebuild -project SwiftLap.xcodeproj -scheme SwiftLap \
  -destination 'platform=iOS Simulator,name=iPhone 17' \
  -configuration Debug CODE_SIGNING_ALLOWED=NO build

# Build the watch app
xcodebuild -project SwiftLap.xcodeproj -scheme SwiftLapWatch \
  -destination 'platform=watchOS Simulator,name=Apple Watch Series 11 (46mm)' \
  -configuration Debug CODE_SIGNING_ALLOWED=NO build

# Run + screenshot on the iOS simulator
xcrun simctl bootstatus <iphone-udid> -b
xcrun simctl install booted <DerivedData>/.../SwiftLap.app
xcrun simctl launch booted com.swiftlap.ios
xcrun simctl io booted screenshot /tmp/shot.png
```

## Auth

- Email login/signup â†’ backend `/api/auth/*` (signup auto-logs-in).
- Google â†’ Supabase Swift SDK `signInWithOAuth(.google)`; Apple â†’ native `SignInWithAppleButton` + `signInWithIdToken`. Both exchange the token via backend `/api/auth/oauth-sync` (role prompt for first-time OAuth users).
- **API auth tokens (since 2026-05-24):** the backend now requires a Bearer token on every `/api` route. `AuthManager` captures the session token on email/OAuth login, hands the session to the Supabase SDK (for auto-refresh), and sets `APIClient.tokenProvider`, which attaches the header to every request (incl. multipart upload). The watch leaves `tokenProvider` nil â€” it only calls the public `/watch/*` device endpoints.
